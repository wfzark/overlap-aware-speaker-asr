# Skill 06: GitHub Demo Excellence

## What question does this skill explore?

How can the project be presented so that the repository is visually clear, memorable, and easy to understand at a glance?

## Why is it relevant to the current project?

The technical baseline is done, so the repository should now communicate its story well to humans who are skimming the project quickly.

## Challenge level

Level 2: Engineering Demo

## Minimum viable attempt

- Improve README structure
- Add a simple architecture diagram
- Add a clear result table and links to summaries

## Stretch goal

- Add a Streamlit dashboard
- Add a demo GIF
- Add a social preview image
- Make the GitHub landing page feel polished and coherent

## Failure is useful if...

- The UI is not perfect, but the structure makes the project easier to understand
- The demo shows why the project matters even if it is not flashy
- The polish work clarifies the research story for outsiders

## Inputs

- Project summaries
- Figures
- Result tables
- Repository structure

## Outputs

- GitHub README hero section
- Demo UI
- Presentation-ready screenshots
- Onboarding guide

## What not to do

- Do not build decoration without explanation.
- Do not bury the main findings under visual clutter.
- Do not make the demo more complex than the research story requires.

## Success criteria

- A new visitor can understand the project in under a minute.
- The README, demo, and figures all tell the same story.
- The visual polish supports the research narrative.

## Suggested agent prompt

Make the repository visually clear and easy to onboard. Focus on the story, the figures, and the decision points rather than on decoration.

## Owner suggestion

Demo / visualization / presentation owner.
